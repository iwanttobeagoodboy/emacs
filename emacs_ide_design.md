# Emacs IDE 架构与配置方案设计（Emacs 30+）

本方案旨在构建一个现代化、高性能、可扩展且长期可维护的 Emacs
开发环境，覆盖代码编辑、项目管理、调试、AI 辅助及知识管理等完整工作流。

------------------------------------------------------------------------

## 一、设计原则

### 1. 内建优先与轻量化

优先使用 Emacs 30
内置能力，减少外部依赖与臃肿框架，提高稳定性与启动速度：

-   eglot（LSP）
-   flymake（诊断）
-   project.el（项目管理）
-   xref（代码导航）
-   treesit（语法解析）
-   package-vc（包管理）

### 2. 模块解耦与职责单一

各系统职责明确，避免功能重叠：

-   补全系统 ≠ AI 补全 ≠ 代码片段
-   搜索系统 ≠ 文件浏览
-   项目管理 ≠ UI 展示

### 3. 高效交互优先

统一通过 Minibuffer 进行交互，实现高效一致体验。

### 4. 分层与渐进式补全

-   精确补全：corfu
-   语义补全：eglot
-   AI 补全：copilot

### 5. 性能优化

异步优先、延迟加载、避免阻塞。

------------------------------------------------------------------------
